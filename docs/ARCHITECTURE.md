# Архитектура проекта



Проект построен по модульному принципу.

основной файл - **main.py**, отвечает за вход в приложение, в нём и находится основной цикл игры. Игра однопоточная, возможно получится сделать на нескольких потоках, но это мечты.



 

### Система сохранения



###### Хранится в /data , управляется в /data/data.py



* ###### savegame.json - сохранение прогресса

###### 

* ###### config.toml - конфигурация игры

###### 

* ###### setting.json - пользовательские настройки



### Пользовательский интерфейс

* ###### level.py - выбор уровней
* ###### menu.py - самописные виджеты
* ###### option.py - изображение и мелкие функции
* ###### set.py - настройки



### Спрайты

* ###### arrow.py - предмет на курсоре
* ###### bullet.py - снаряды
* ###### map.py - спрайты карты
* ###### window.py - битое стекло или шипы
* ###### zombie.py - все зомби



##### config.toml

**Враги:**

1. скорость
2. здоровье
3. урон от шипов
4. урон от снарядов
5. счётчик(для создания волн)





### Карта



Представлен двумерным массивом nuppy, x и y
размер карты: 96 x 50 клеток(игровое поле меньше из за нижнего меню)



0 - пусто, только спрайт карты есть
1 - резерв
2 - турель
3 - шипы
